# 关键页面设计说明 / 线框需求

## 1. 设计目标

本项目 PoC 的设计重点不是做复杂视觉，而是快速搭建一套适合企业 AI 工作流的操作台，使用户能：
- 看懂系统做了什么
- 快速处理异常
- 清楚知道下一步该做什么
- 对自动动作保有控制权

## 2. 设计原则

1. **状态优先于装饰**
2. **列表 + 详情双栏为主**
3. **异常优先呈现**
4. **自动决策必须有解释**
5. **人工复核路径必须短**
6. **高风险动作必须有确认门**

## 3. 页面清单

### Page 1：总览 Dashboard
**目的：** 给会计经理 / 会计专员快速掌握当前运行状态。

**建议模块：**
- 今日新邮件数量
- 今日新凭证数量
- 待复核数量
- 本月凭证覆盖率
- 高风险异常数量
- 待催票客户数
- 月结 checklist 预警数

**关键交互：**
- 点击卡片跳转列表
- 按客户筛选
- 按状态筛选

---

### Page 2：新邮件与凭证处理工作台（核心页）
**目的：** 处理 M8 主链路。

**布局建议：**
- 左侧：任务列表
- 中间：邮件与附件预览
- 右侧：OCR 字段结果 / Xero 映射 / 操作按钮

**列表字段建议：**
- 邮件时间
- 客户名
- 附件数
- 附件类型
- OCR 状态
- 当前处理状态
- 风险等级

**详情区字段建议：**
- 邮件主题
- 发件人
- 附件缩略信息
- OCR 字段表
- 置信度
- 建议动作

**按钮建议：**
- 确认并写入 Xero
- 编辑字段
- 标记异常
- 稍后处理

---

### Page 3：人工复核页
**目的：** 集中处理低置信度任务。

**需要展示：**
- 原附件预览
- OCR 结果
- 系统置信度
- 缺失字段提示
- 历史相似记录 [待验证]
- 修改前 / 修改后值

---

### Page 4：缺票检索与催票页
**目的：** 处理 M9 + M2。

**列表区：**
- 客户
- 缺失月份
- 缺失凭证类型
- 最近催票时间
- 当前分级
- 检索状态

**详情区：**
- 缺失说明
- 历史邮件命中结果
- 候选附件
- 系统建议邮件正文
- 审核/发送动作

**交互：**
- 选中候选历史邮件作为证据
- 选择 L1/L2/L3 模板
- 编辑后发送
- 记录结果

---

### Page 5：月结清单页
**目的：** 处理 M3。

**内容：**
- 客户清单
- checklist 总完成度
- 即将到期项
- 已逾期项
- 责任角色
- 最近更新时间

---

### Page 6：审计与日志页
**目的：** 提升企业级可信度。

**应展示：**
- 处理时间线
- 自动动作
- 人工动作
- 字段修改记录
- 外发记录
- 错误与重试记录

## 4. 核心组件

| 组件 | 用途 |
|---|---|
| 状态标签 | 待处理 / 待复核 / 已写入 / 异常 / 已发送 |
| 置信度条 | 展示 OCR / 检索置信度 |
| 证据卡片 | 展示历史邮件命中证据 |
| 比对表格 | 展示原值与修改值 |
| 风险提示框 | 展示高风险外发或关键缺失 |
| 操作确认弹窗 | 高风险动作二次确认 |

## 5. 页面优先级

| 页面 | 优先级 |
|---|---:|
| 新邮件与凭证处理工作台 | P0 |
| 人工复核页 | P0 |
| 缺票检索与催票页 | P0 |
| Dashboard | P0 |
| 月结清单页 | P0 |
| 审计与日志页 | P1 |

## 6. 设计待验证项

- 是否需要多语言界面
- 附件预览格式支持范围
- 是否需要共享邮箱视图
- 是否默认展示客户财务敏感字段
- 是否需要批量处理交互
