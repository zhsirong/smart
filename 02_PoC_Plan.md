# PoC 范围与里程碑计划

## 1. 文档目标

定义 AG-B3-01 的 PoC 范围、里程碑、资源要求、交付节奏和验收门禁。

## 2. PoC 定位

本次 PoC 为 **Proof of Concept / 概念验证**，目标不是一次性做完全部能力，而是验证以下三件事：
1. 核心链路是否可以打通
2. 指标是否有明显改进空间
3. 业务团队是否接受 AI 介入方式

## 3. PoC 范围

### 3.1 PoC 纳入范围
- M8：邮件凭证自动处理
- M9：历史邮件检索
- M1：凭证状态监控
- M2：邮件催票
- M3：月结清单追踪

### 3.2 PoC 不纳入范围
- M4 GST 自动判责闭环
- M5 高准确率预测建模
- M6 自动报表发送正式上线
- 多国家流程适配
- 高风险无审核外发

## 5. 周计划

### Week 1-2：前置依赖解除
- 确认邮箱 API 类型与授权方式
- 确认 file.ai workspace / schema / API key / webhook
- 确认 Xero tenant 与写权限
- 获取月结清单模板
- 确认试点客户名单

### Week 3-4：M8 + M1
- 邮件读取
- 附件分类
- OCR 触发
- Xero 回写
- 凭证状态看板

### Week 5-6：M9 + M2
- 历史邮件检索
- 催票规则引擎
- 催票邮件生成与发送/待审
- 催票记录回写

### Week 7-8：M3 + 联调验收
- 月结清单追踪
- 全链路联调
- UAT
- KPI 初步验证

## 6. 关键门禁

| 阶段 | 门禁 |
|---|---|
| Week 2 | DR-01 / DR-02 / DR-03 至少形成明确方案，否则转为半人工模式 |
| Week 4 | M8 端到端跑通至少 1 条真实链路 |
| Week 6 | M9+M2 对至少 1 家客户完成缺票检索与催票闭环 |
| Week 8 | E2E 用例通过；A-01 / A-02 / A-06 达到阶段目标 |

## 9. 交付成功标准

| 项目 | 通过标准 |
|---|---|
| 核心功能 | M8/M9/M1/M2/M3 跑通 |
| 业务体验 | 会计专员可在单一工作台处理主要任务 |
| 指标 | A-01、A-02、A-06 达到阶段目标 |
| 风险控制 | 审计、权限、人工 gate 清晰 |
| 业务接受度 | 会计经理认可继续推进 |

## 10. 降级方案

若 Week 2 前接口未完全就绪，则采用如下降级方案：
- 邮箱侧改为人工转发至指定邮箱 / 上传至指定入口
- file.ai 改为人工触发但系统记录状态
- Xero 改为待提交清单导出，不直接写入
- 自动催票改为“生成草稿 + 人工发送”
