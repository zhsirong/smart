# KPI 测量方案

## 1. 目标

建立 PoC 阶段 KPI 的定义、口径、采集方式和验收标准，避免“看起来有效但无法量化”。

## 2. 核心 KPI

### A-01 月结周期
- 定义：从月结启动到客户确认 / 内部完成的总天数
- 当前基线：10-15 天（待客户确认精确值）
- PoC 目标：≤10 天
- 数据来源：月结任务记录 / checklist / 客户确认记录 [待验证]
- 采集频率：每月

### A-02 催票响应率
- 定义：发送催票邮件后，在规定窗口内收到有效回复的比例
- 当前基线：人工催票（待补充真实比例）
- PoC 目标：≥70%
- 数据来源：催票记录 + 邮件回复记录
- 采集频率：每周 / 每月

### A-03 月结准时完成率
- 定义：在约定 deadline 前完成月结的客户比例
- 当前基线：待补充
- PoC 目标：≥80%
- 数据来源：客户月结记录 + deadline 配置
- 采集频率：每月

### A-06 凭证覆盖率
- 定义：月结前所需凭证中已收齐并确认的比例
- 当前基线：未量化
- PoC 目标：≥90%
- 数据来源：M1 状态表 + checklist 口径
- 采集频率：每日 / 每周 / 月结前

## 3. 辅助 KPI

| KPI | 定义 | 状态 |
|---|---|---|
| OCR 自动通过率 | 无需人工复核的 OCR 任务占比 | 可测 |
| OCR 准确率 | 结构化字段正确率 | 可测 |
| Xero 写入成功率 | 成功回写 / 尝试回写 | 可测 |
| 检索命中率 | 历史邮件检索成功定位附件的比例 | 可测 |
| 催票自动发送占比 | 无人工干预发送比例 | 待定义是否启用 |
| 异常率 | 失败 / 异常任务占比 | 可测 |

## 4. 口径说明

### 4.1 “有效回复”定义
[待验证]
建议定义为：
- 客户回复邮件且内容与缺票事项相关
- 或客户补发所需附件

### 4.2 “月结完成”定义
[待验证]
建议明确：
- 以内部完成为准
- 或以客户确认收到报表为准
- 或两者分开统计

### 4.3 “凭证覆盖率”分母定义
[待验证]
建议明确：
- 以 checklist 所需凭证项为分母
- 不以系统已识别任务数为分母

## 5. 数据采集方案

| 指标 | 数据表 | owner | 备注 |
|---|---|---|---|
| A-01 | checklist / project table | PM / Ops | 需定义起止点 |
| A-02 | chase record + email reply | Product / Ops | 需识别有效回复 |
| A-03 | closing record | Business Owner | 需统一 deadline |
| A-06 | voucher state table | Product / Ops | 可日常滚动统计 |

## 6. 看板建议

### 管理层看板
- 月结周期
- 准时完成率
- 覆盖率
- 试点客户对比

### 运营/会计看板
- 今日待复核
- 今日失败回写
- 当前缺票客户
- 本周催票回复情况

## 7. 验收建议

PoC 结束时至少输出：
- 指标对比表（baseline vs PoC）
- 样本量说明
- 数据口径说明
- 风险与偏差解释

## 8. 待验证项

- baseline 的真实数据
- 数据采集字段是否已存在
- 是否能自动识别“有效回复”
- A-03 与 A-04 是否合并口径
