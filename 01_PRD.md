## 2. 项目背景

InCorp 新加坡会计团队在月结前端处理中存在明显的人工操作瓶颈：
- 每天需要处理大量客户邮件及其附件
- 需手动识别 invoice / 银行流水 / 收据等附件类型
- 需手动上传 file.ai 做 OCR
- 需人工核对结构化结果并录入 Xero
- 月结时若发现缺票，还需翻查大量历史邮件并手动发起催票

这导致月结流程高度依赖人工、效率低、标准化弱，并占用会计经理与会计专员大量时间。

## 3. 产品目标

### 3.1 终极目标
将新加坡会计团队的月结前端处理从“邮件手工处理 + 凭证手工录入 + 历史邮件人工翻查催票”转变为“AI 自动识别与处理邮件凭证 + 智能检索历史邮件 + 自动催票 + 实时状态追踪”。

### 3.2 PoC 目标
在 6 周 PoC 周期内，打通最小闭环：
- M8 邮件凭证自动处理
- M9 历史邮件智能检索
- M1 凭证状态追踪
- M2 邮件催票执行
- M3 月结清单追踪

### 3.3 成功定义
- 月结周期压缩至 ≤10 天
- 凭证覆盖率达到 ≥90%
- 催票响应率达到 ≥70%
- 关键流程可在试点客户上稳定运行

## 4. 用户角色

| 角色 | 描述 | 核心诉求 |
|---|---|---|
| role_b3_accountant | 会计专员 | 减少手工处理邮件、上传 OCR、录入 Xero、翻历史邮件 |
| role_b3_senior | 高级会计师 / CPA | 保留专业复核权，降低低价值重复工作 |
| role_b3_account_manager | 客户沟通/报表发送角色 | 减少重复答疑与标准报表发送负担 |
| InCorp IT / 技术团队 | 接口与权限 owner | 明确 API、认证、审计与稳定性要求 |
| 会计经理 | 业务 owner | 缩短月结周期，提升准时完成率和团队产能 |

## 5. 核心痛点

1. **邮件入口未结构化**：发票、流水等混杂在邮件系统中
2. **处理链路过长**：下载、上传、核对、录入四段式人工链路
3. **缺票检索成本高**：月结时需反复翻查历史邮件
4. **催票标准不稳定**：L1/L2/L3 分级更多依赖个人经验
5. **月结状态不透明**：凭证覆盖率、异常项、未完成项无法实时追踪
6. **关键节点缺少系统化审核门**：GST、低置信 OCR、重要外发邮件需明确控制

## 6. 产品范围

### 6.1 In Scope（PoC）
- 邮件凭证自动识别与抓取
- OCR 触发与结构化字段回填
- Xero 凭证写入
- 低置信度人工复核
- 凭证覆盖率看板
- 历史邮件检索
- 自动生成催票邮件
- 月结清单自动追踪

### 6.2 Out of Scope（PoC）
- 复杂财务分析与预算管理
- 多司法辖区月结规则
- 法务函件 / 正式催款函自动发送
- 全自动无审核的高风险财务判断
- 报表智能解读客服化能力（可作为后续扩展）

## 8. 核心用户故事

### 8.1 M8
- 作为会计专员，我希望系统自动识别客户邮件中的 invoice 和银行流水附件，以减少手工下载和分类。
- 作为会计专员，我希望 OCR 结果能自动进入待确认队列，让我只处理低置信度异常。
- 作为会计专员，我希望系统将确认后的结构化数据写入 Xero，减少重复录入。

### 8.2 M9 + M2
- 作为会计专员，我希望系统在发现缺票时自动检索历史邮件，减少人工翻查。
- 作为会计经理，我希望系统根据逾期程度自动生成催票邮件，提升响应率。

### 8.3 M1 + M3
- 作为会计经理，我希望看到某客户当前凭证覆盖率、缺失凭证与月结清单状态，便于提前干预。

## 9. 关键流程摘要

1. 收到客户邮件
2. 系统解析附件并分类
3. 将可识别文件推送到 file.ai
4. file.ai 返回 OCR 结构化结果
5. 根据置信度自动写入 Xero或进入人工复核
6. 更新凭证状态与覆盖率
7. 若覆盖率不足，触发历史邮件检索
8. 生成催票邮件并发送 / 待审核
9. 月结清单持续追踪并预警

## 10. 核心需求

### 10.1 功能性需求
- 支持从邮箱读取消息、附件元数据和附件正文
- 支持识别 invoice / 银行流水 / 收据 / 其他附件类型
- 支持触发 OCR 解析并接收结构化结果
- 支持结构化字段人工修订
- 支持将凭证数据写入 Xero
- 支持记录每条凭证状态
- 支持历史邮件检索与相似附件定位
- 支持催票邮件生成、发送、记录
- 支持月结 checklist 状态维护与预警

### 10.2 非功能性需求
- 支持审计日志
- 支持角色权限控制
- 支持人工接管
- 支持失败重试与幂等
- 支持关键节点状态可追踪
- 支持异常告警

## 13. 风险

- 邮箱 API 无法开通，导致 M8/M9 无法自动拉取邮件
- file.ai API 不可用，导致 OCR 只能人工触发
- Xero 无写权限，导致 PoC 只能做到“待提交”状态
- 历史邮件中附件覆盖率不足，削弱 M9 的价值
- 自动外发催票邮件存在合规与客户体验风险
