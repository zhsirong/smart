# 决策规则与人工审核门

## 1. 文档目标

明确系统在什么条件下自动执行、什么条件下进入人工复核、什么条件下禁止自动执行。

## 2. 总体原则

1. 低风险、高重复动作优先自动化
2. 高风险、低确定性动作必须有人审
3. 所有自动动作必须支持追踪与回滚
4. 所有外发动作默认建议保守控制

## 3. M8 决策规则

### 3.1 附件分类
- 若附件类型置信度 ≥ [待验证阈值]，自动进入对应处理流
- 若附件类型不明，标记为 `other` 并进入人工队列

### 3.2 OCR 结果处理
- 若 confidence_score ≥ [待验证，例如 0.85] 且关键字段齐全，则自动进入 Xero 写入
- 若 confidence_score < 阈值，则进入人工复核
- 若缺少 invoice_number / amount / date 等关键字段，则禁止自动写入

### 3.3 Xero 写入
- 若命中重复凭证，则禁止自动新建并提示人工确认
- 若 tax / account code 映射失败，则进入人工复核
- 若 Xero 接口失败，则记录失败并进入重试队列

## 4. M9 检索规则

- 默认以客户 + 周期 + 文件类型为主检索条件
- 若命中多个候选，展示前 N 条并附理由
- 若检索无明显结果，不自动宣称“找到原始发票”
- 若历史邮件来源不可信 / 线程不完整，则标记“低证据”

## 5. M2 催票规则

### 5.1 分级规则
- L1：轻提醒 [待验证]
- L2：重复提醒 [待验证]
- L3：升级提醒 [待验证]

### 5.2 自动发送规则建议
| 条件 | 建议动作 |
|---|---|
| 首次催票、低风险客户 | 可生成草稿或自动发送 [待验证] |
| 高敏感客户 / 高层联系人 | 强制人工审核 |
| 系统未命中历史证据 | 仅生成保守催票文案 |
| 一周内已发送过同类催票 | 不自动重复发送 |

### 5.3 文案约束
- 不使用威胁性或法务措辞
- 不承诺未核实的金额或结论
- 不暴露内部判断逻辑
- 保持专业、简洁、礼貌

## 6. M3 checklist 规则

- 截止前 5/3/1 天自动提醒
- 逾期后升级提醒到指定角色 [待验证]
- 若前置凭证覆盖率不足，则阻止部分后续节点自动完成

## 7. M4 GST 规则（预留）

- 若税率与规则库不一致，强制 CPA 复核
- 若跨期 / 特殊税码场景出现，禁止自动通过

## 8. Human Gates 一览

| Gate | 触发条件 | 责任角色 |
|---|---|---|
| G1 OCR 复核 | OCR 低置信度 / 缺关键字段 | 会计专员 |
| G2 凭证去重确认 | 疑似重复凭证 | 会计专员 |
| G3 Xero 关键映射确认 | tax/account code 不确定 | 会计专员 / 高级会计师 |
| G4 高风险催票审核 | 高等级催票 / 高敏感客户 | 会计经理 [待验证] |
| G5 GST 异常审核 | 税务规则异常 | 高级会计师 / CPA |
| G6 报表发送审核 | 正式对客报表外发 | 经理 / Account Manager [待验证] |

## 9. 审计要求

- 记录自动决策输入、规则版本、输出结果
- 记录人工修改前后值
- 记录最终发送版本
- 记录失败与重试轨迹

## 10. 待验证项

- 各阈值数值
- 自动发送边界
- 升级链路
- 不同客户层级的 gate 差异
