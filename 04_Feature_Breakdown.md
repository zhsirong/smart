# 功能拆解与模块需求清单

## 1. 说明

本文件将 SRP 中的 M1 / M2 / M3 / M8 / M9 等模块拆解为可执行的需求点，用于研发评估、排期和任务切分。

---

## 2. M8 邮件凭证自动处理引擎（P0）

### 2.1 目标
自动完成从“收到邮件”到“凭证写入 Xero”的主链路，减少手工识别、下载、上传、核对和录入。

### 2.2 子功能

| 编号 | 子功能 | 说明 | 优先级 |
|---|---|---|---:|
| M8-01 | 邮件拉取 | 读取指定邮箱中的新邮件 | P0 |
| M8-02 | 附件识别 | 识别 invoice / bank statement / receipt / other | P0 |
| M8-03 | 文件预处理 | 提取附件元数据、标准化命名、去重 | P0 |
| M8-04 | OCR 触发 | 调用 file.ai 进行结构化抽取 | P0 |
| M8-05 | 结果回收 | 接收 OCR 结构化结果 | P0 |
| M8-06 | 置信度判断 | 根据阈值决定自动通过或人工复核 | P0 |
| M8-07 | Xero 写入 | 创建或更新凭证 | P0 |
| M8-08 | 状态回写 | 更新任务状态与错误信息 | P0 |
| M8-09 | 审计日志 | 记录处理过程 | P1 |

### 2.3 输入
- 邮件 ID
- 发件人
- 客户标识 [待验证]
- 附件文件

### 2.4 输出
- 附件分类结果
- OCR 结构化结果
- Xero 凭证 ID [待验证]
- 处理状态
- 异常原因

### 2.5 异常处理
- 附件不可读
- OCR 超时
- OCR 结果缺关键字段
- Xero 写入失败
- 重复凭证

## 3. M1 凭证状态监控引擎（P0）

### 3.1 目标
将凭证状态透明化，支持月结覆盖率跟踪。

### 3.2 子功能
- M1-01 状态模型定义（待处理/待复核/已写入/失败/缺失）
- M1-02 客户维度覆盖率统计
- M1-03 时间维度统计（日/周/月）
- M1-04 缺失凭证列表
- M1-05 异常项列表
- M1-06 dashboard 展示

---

## 4. M9 历史邮件智能检索（P0）

### 4.1 目标
在发现缺票时，自动从历史邮件中寻找最相关的邮件与附件。

### 4.2 子功能
- M9-01 历史邮件索引建立
- M9-02 历史附件索引建立
- M9-03 基于客户 + 日期 + 金额 + 关键词的检索
- M9-04 相似附件候选列表
- M9-05 检索证据展示
- M9-06 一键生成催票上下文

### 4.3 检索输入
- 客户名 / 客户 ID
- 月结周期
- 缺失凭证类型
- 参考金额 [待验证]
- 参考日期 [待验证]

### 4.4 检索输出
- 候选邮件
- 候选附件
- 匹配理由
- 建议催票对象

## 5. M2 邮件分级催票执行器（P0）

### 5.1 目标

### 5.2 子功能

- M2-02 L1/L2/L3 分级
- M2-03 催票模板管理
- M2-04 邮件正文生成
- M2-05 邮件发送/草稿生成
- M2-06 发送记录回写
- M2-07 回复状态追踪 [待验证]

### 5.3 关键待定规则
- L1/L2/L3 的时间阈值 [待验证]
- 是否默认人工审核 [待验证]
- 抄送规则 [待验证]
- 催票语气合规要求 [待验证]
